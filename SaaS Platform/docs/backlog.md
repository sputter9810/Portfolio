# User Stories – Momentum

## 🧑‍💻 Authentication

* As a user, I want to register an account so that I can access my own workspace.
* As a user, I want to log in securely so that my data is protected.
* As a user, I want to log out so that my account remains secure on shared devices.

---

## 📋 Task Management

* As a user, I want to create tasks so that I can keep track of what I need to do.
* As a user, I want to edit tasks so that I can update details as plans change.
* As a user, I want to delete tasks so that I can remove irrelevant work.
* As a user, I want to set due dates so that I can manage deadlines.
* As a user, I want to assign priorities to tasks so that I can focus on what matters most.
* As a user, I want to update task status so that I can track my progress.

---

## 📅 Calendar Events

* As a user, I want to create calendar events so that I can schedule commitments.
* As a user, I want to edit events so that my schedule stays accurate.
* As a user, I want to delete events so that outdated commitments are removed.
* As a user, I want to view events in a calendar so that I can see my upcoming schedule.

---

## ⏱️ Focus Sessions

* As a user, I want to create focus sessions so that I can plan dedicated work time.
* As a user, I want to link sessions to tasks so that my work has clear purpose.
* As a user, I want to update session status so that I can track completion.
* As a user, I want to edit sessions so that I can adjust my plans.

---

## 📂 Resource Management

* As a user, I want to upload files so that I can store resources related to my work.
* As a user, I want to view my uploaded resources so that I can access them easily.
* As a user, I want to delete resources so that I can manage storage.
* As a user, I want to link resources to tasks or sessions so that everything relevant is connected.

---

## 📊 Dashboard

* As a user, I want to see an overview of my tasks so that I know what needs attention.
* As a user, I want to see upcoming sessions so that I can plan my time.
* As a user, I want to view upcoming events so that I stay aware of commitments.

---

## 🧠 System Behaviour

* As a user, I want my data to be private so that only I can access my information.
* As a user, I want the system to handle errors gracefully so that I can recover from mistakes.

---

## ⭐ Nice-to-Have (Post-MVP)

* As a user, I want reminders so that I don’t miss deadlines.
* As a user, I want suggestions for scheduling so that I can optimise my time.
* As a user, I want analytics so that I can track productivity trends.

---

## 📌 Summary

These user stories define the functional expectations of the MVP and guide development priorities.

They ensure the system is built around **real user needs**, not just technical implementation.
